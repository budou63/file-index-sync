# file-index-sync

## プロジェクト概要
`新ファイル基準表.xlsx` を中心に、CSV 整形・コードマスタ補完・差分インポート CSV 生成までを一貫して管理するためのリポジトリです。

## 目的
- ローカル作業フォルダの成果物（Excel ブック、マクロ、CSV、ドキュメント）を Git/GitHub で履歴管理する
- 定型作業の手順を明文化し、再現性を上げる
- 差分インポートに必要な CSV を安定して作成できる状態を保つ

## 想定フロー
1. CSV 出力
2. `手順1_目録CSVを取り込む` で CSV を選択し、`目録CSV` シートへ上書き取込
3. `手順2_新ファイル基準表を作成する` で `新ファイル基準表` を生成
4. `コード管理CSV` を参照してコード補完
5. 差分インポート CSV 生成

## シートの役割
- `目録CSV`: 元CSV保持用（取込のみ。整形しない）
- `目録CSV` の想定構造:
  1行目=正式見出し、2～10行目=補助情報、11行目以降=実データ
- `新ファイル基準表`: 表示用の整形シート（`分類名１/２/３, タイトル, 保存期間, 媒体種別` を出力）
- `コード管理CSV`: コード対応管理
- `差分インポートCSV`: インポート用出力

## 利用者向けマクロ
- `一括_取込から新ファイル基準表作成まで`: 通常運用で使う推奨マクロ（CSV取込 → 新ファイル基準表作成）
- `手順1_目録CSVを取り込む`: CSV ファイルを選択し、Shift-JIS(932) 前提で `目録CSV` シートへ上書き取込
- `手順2_新ファイル基準表を作成する`: `目録CSV` から必要6列を `新ファイル基準表` へ整形出力
- 通常は「1. CSV取込 → 2. 新ファイル基準表作成」の順で使用

## 管理対象
- Excel ブック（`.xlsx`, `.xlsm` など）
- CSV ファイル
- README / ドキュメント
- マクロ関連資産

## 補足
一時ファイル（`~$*.xls*`, `*.tmp`, `Thumbs.db`, `.DS_Store` など）は `.gitignore` で除外しています。
