# file-index-sync

## プロジェクト概要
`新ファイル基準表.xlsx` を中心に、CSV 整形・コードマスタ補完・差分インポート CSV 生成までを一貫して管理するためのリポジトリです。

## 目的
- ローカル作業フォルダの成果物（Excel ブック、マクロ、CSV、ドキュメント）を Git/GitHub で履歴管理する
- 定型作業の手順を明文化し、再現性を上げる
- 差分インポートに必要な CSV を安定して作成できる状態を保つ

## 想定フロー
1. CSV 出力
2. `手順1_目録CSVを取り込む` で CSV を選択し、`目録CSV` シートへ上書き取込
3. `手順2_新ファイル基準表を作成する` で `新ファイル基準表` を生成
4. `手順3_コード管理CSVを作成する` で `コード管理CSV` のコード/表示名マスタを抽出
5. `コード管理CSV` を参照してコード補完
6. `手順4_差分インポートCSVを作成する` で更新行と新規追加行を抽出
7. 差分インポート CSV 生成

## シートの役割
- `目録CSV`: 元CSV保持用（取込のみ。整形しない）
- `目録CSV` の想定構造:
  1行目=見出し、2行目以降=データ
- `新ファイル基準表`: 表示用の整形結果シート（`目録CSV` の全列をコピー後、指定列を左から順に並べ替えて表示。指定外列は削除せず非表示で保持）
  - ファイルID
  - ファイル番号
  - バーコード番号
  - 毎年度利用区分1:毎年作成 2:単年のみ
  - 原課
  - 年度（和暦）
  - 分類名１
  - 分類名２
  - 分類名３
  - タイトル
  - 保存期間
  - 媒体種別
- `コード管理CSV`: コード対応管理
  - `目録CSV` から抽出した「区分 / 表示名 / コード」の対応マスタを保持
- `差分インポートCSV`: インポート用出力

## 利用者向けマクロ
- `一括_取込から新ファイル基準表作成まで`: 通常運用で使う推奨マクロ（CSV取込 → 新ファイル基準表作成）
- `手順1_目録CSVを取り込む`: CSV ファイルを選択し、Shift-JIS(932) 前提で `目録CSV` シートへ上書き取込
  - 取込直後に見出し行（1行目）の改行コード（vbCrLf / vbCr / vbLf）を除去して見出し一致を安定化
- `手順2_新ファイル基準表を作成する`: `目録CSV` の全列コピー後、指定12列を左側に並べ替え。キー列（ファイルID/ファイル番号/バーコード番号）は列を保持したまま非表示にし、編集対象列のみ表示（存在しない見出しはスキップ。他列は非表示、削除しない）。「毎年度利用区分1:毎年作成 2:単年のみ」は 1→毎年作成、2→単年のみ に表示変換
- `手順3_コード管理CSVを作成する`: `目録CSV` からコード⇔表示名を抽出（項目名/コード/表示名）し、固定選択肢マスタ（毎年度利用区分、年度暦年区分、完結区分、公開区分、歴史資料区分、廃棄区分、履歴ファイル区分、機密ファイル区分）も重複除外して `コード管理CSV` に出力
- `手順4_差分インポートCSVを作成する`: `目録CSV` と `新ファイル基準表` をキー（ファイルID/ファイル番号/バーコード番号）で比較し、更新行と条件一致する新規追加行のみを固定見出しレイアウトで `差分インポートCSV` へ出力（同名見出しがない列は空白。`所管所属` は `原課ID`、`毎年度利用区分` は `毎年度利用区分1:毎年作成 2:単年のみ` から取得）
- 通常は「1. CSV取込 → 2. 新ファイル基準表作成」の順で使用

## 管理対象
- Excel ブック（`.xlsx`, `.xlsm` など）
- CSV ファイル

## 新ファイル基準表のID自動補完（Worksheet_Change）設定
- `新ファイル基準表` で表示名入力時にID列へ自動補完するには、**シートモジュール**に `Worksheet_Change` が必要です。
- 貼り付け先: `VBAProject > Microsoft Excel Objects > 新ファイル基準表`
- コードは `新ファイル基準表_SheetModule.bas` を使用してください（標準モジュールでは発火しません）。

```vb
Private Sub Worksheet_Change(ByVal Target As Range)
    On Error GoTo ExitHandler
    Application.EnableEvents = False

    新ファイル基準表_表示名からID自動補完 Target

ExitHandler:
    Application.EnableEvents = True
End Sub
```

### 動かないときの確認
1. 上記 `Worksheet_Change` が「新ファイル基準表」シートモジュールに存在するか
2. 標準モジュール `Module1` に `Public Sub 新ファイル基準表_表示名からID自動補完(ByVal Target As Range)` が存在するか
3. 即時ウィンドウで `Application.EnableEvents = True` を実行し、イベント無効化が残っていないか
4. `コード管理CSV` の構成が `A列=項目名 / B列=コード / C列=表示名` になっているか
