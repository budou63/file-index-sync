# file-index-sync

## プロジェクト概要
`新ファイル基準表.xlsx` を中心に、CSV 整形・コードマスタ補完・差分インポート CSV 生成までを一貫して管理するためのリポジトリです。

## 目的
- ローカル作業フォルダの成果物（Excel ブック、マクロ、CSV、ドキュメント）を Git/GitHub で履歴管理する
- 定型作業の手順を明文化し、再現性を上げる
- 差分インポートに必要な CSV を安定して作成できる状態を保つ

## 想定フロー
1. CSV 出力
2. `手順1_目録CSVを取り込む` で CSV を選択し、`目録CSV` シートへ上書き取込
3. `手順2_目録CSVの表示を整える` で必要列だけを表示向けに整理
4. `手順3_旧基準表を作成する` で `新ファイル基準表` を生成
5. `コード管理CSV` を参照してコード補完
6. 差分インポート CSV 生成

## シートの役割
- `目録CSV`: 元データ（取込元）
- `目録CSV` の想定構造:
  1行目=正式見出し、2～10行目=補助情報、11行目以降=実データ
  必要列のみ左側に並べ替え、その他列を非表示にする表示調整が可能
- `新ファイル基準表`: 閲覧・編集用の表示シート（整形結果）
- `コード管理CSV`: コード対応管理
- `差分インポートCSV`: インポート用出力

## 利用者向けマクロ
- `一括_取込から旧基準表作成まで`: 通常運用で使う推奨マクロ（取込 → 表示整理 → 旧基準表作成を順に実行）
- `手順1_目録CSVを取り込む`: CSV ファイルを選択し、Shift-JIS(932) 前提で `目録CSV` シートへ上書き取込
- `手順2_目録CSVの表示を整える`: `分類名１/２/３, タイトル, 保存期間, 媒体種別` を左から並べ、その他列を非表示
- `手順3_旧基準表を作成する`: `目録CSV` から `新ファイル基準表` へ整形（上記構造を前提）
- 必要に応じて単機能マクロ（手順1〜3）を個別実行

## 管理対象
- Excel ブック（`.xlsx`, `.xlsm` など）
- CSV ファイル
- README / ドキュメント
- マクロ関連資産

## 補足
一時ファイル（`~$*.xls*`, `*.tmp`, `Thumbs.db`, `.DS_Store` など）は `.gitignore` で除外しています。
